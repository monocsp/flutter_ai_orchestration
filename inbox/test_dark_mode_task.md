# 다크모드 전환 시 일부 화면에서 텍스트가 보이지 않는 문제 수정

## 작업 유형
- 버그 수정

## 작업 배경
- 앱 설정에서 다크모드를 켜면 채팅 목록 화면과 알림 설정 화면에서 텍스트 색상이 배경과 동일해져 내용이 보이지 않는다는 사용자 제보가 접수되었습니다.
- QA팀 확인 결과 iOS/Android 공통으로 재현되며, 시스템 다크모드가 아닌 앱 내 수동 전환 시에만 발생합니다.
- 최근 디자인 시스템 리팩토링(v2.3.0)에서 색상 토큰을 교체한 이후 발생한 것으로 추정됩니다.

## 목표
- 다크모드 전환 시 모든 화면에서 텍스트가 정상적으로 표시되어야 합니다.
- 기존 라이트모드 UI에 영향을 주지 않아야 합니다.

## 사용자 시나리오
1. 사용자가 설정 > 테마에서 다크모드를 수동 선택합니다.
2. 채팅 목록 화면으로 이동합니다.
3. 채팅방 이름, 마지막 메시지 미리보기, 시간 텍스트가 모두 정상 표시됩니다.
4. 알림 설정 화면으로 이동합니다.
5. 토글 레이블과 섹션 헤더 텍스트가 모두 정상 표시됩니다.

## 상세 요구사항
- 하드코딩된 색상값(`Color(0xFF000000)` 등)을 디자인 토큰으로 교체해야 합니다.
- `AppColors.textPrimary`, `AppColors.textSecondary` 등 시맨틱 토큰을 사용해야 합니다.
- 다크모드/라이트모드 전환 시 `hot restart` 없이 즉시 반영되어야 합니다.
- 커스텀 위젯(`ChatListTile`, `NotificationToggleRow`)도 테마 변경에 반응해야 합니다.

## 예상 영향 범위
- 채팅 목록 화면 (`ChatListScreen`)
- 알림 설정 화면 (`NotificationSettingsScreen`)
- 공통 색상 정의 (`AppColors`, `AppTheme`)
- 커스텀 위젯 (`ChatListTile`, `NotificationToggleRow`)
- 테마 전환 로직 (`ThemeProvider` 또는 `ThemeCubit`)

## 완료 조건
- 다크모드에서 채팅 목록의 모든 텍스트가 가독성 있게 표시됩니다.
- 다크모드에서 알림 설정의 모든 텍스트가 가독성 있게 표시됩니다.
- 라이트모드 기존 UI에 변경이 없습니다.
- 시스템 다크모드 연동과 앱 내 수동 전환 모두 정상 동작합니다.

## 확인 필요
- `AppColors` 토큰이 다크모드용 값을 올바르게 정의하고 있는지
- `ChatListTile`과 `NotificationToggleRow`가 다른 화면에서도 재사용되는지
- 디자인 시스템 v2.3.0 변경 시 누락된 매핑이 또 있는지
- `ThemeMode.system` 경로와 `ThemeMode.dark` 수동 경로의 분기가 올바른지
