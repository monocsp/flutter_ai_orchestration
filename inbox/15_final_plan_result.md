## 분석 과정 메모

- **핵심 쟁점**: 원문은 "방향은 맞으니 UX 보정하라"는 피드백인데, 1차 분석이 이를 "감정 분석 파이프라인 구축 프로젝트"로 과대 해석했음. 비판 검토가 이를 정확히 짚었고, 최종 계획은 이 교정을 반영해야 함.
- **해석 갈림길**: ① 변화 요약의 기술적 수준(NLP 엔진 vs self-report 기반 단순 집계), ② 인지·정서 측정의 시점(프로토타입 vs 정식 런칭 후), ③ 상위 목적 문장의 무게(카피 한 줄 vs 제품 정체성 정렬). 세 항목 모두 비판 검토 쪽을 채택함 — 원문 의도에 더 충실하고, 실행 가능성이 높음.
- **확신 부족 부분**: 실제 감정 데이터 스키마를 확인하지 못해, '변화 요약 기본 MVP'의 구현 난이도는 개발팀 확인 전까지 추정치임. 또한 발신자의 정확한 직급/권한은 문서만으로 단정 불가.
- **다음 검토자에게**: 이 계획서는 "기획자가 내일부터 착수 가능한 수준"을 목표로 했으나, 개발팀과의 기술 검증(특히 감정 데이터 구조, 기존 추천 로직)은 첫 3일 내 반드시 병행해야 함. 법무/개인정보 검토도 능력 축적 메시지 표현 확정 전에 선행 필요.

## 1. 최종 판단

- **문서 성격**: UX 개선 + 정책 변경 (복합)
- **최종 판단 요약**: 원문은 돌로무드 재방문 UX의 중심축을 '놀이/수집'에서 '감정 변화 인식'으로 재정렬하라는 경영진 피드백이다. 방향 자체는 유효하므로 전면 재설계가 아닌 UX 보정 + 프로토타입 검증으로 접근한다. 단, self-report 기반 단순 집계로 MVP 범위를 제한하고, 고급 분석(NLP, 시계열 엔진)은 후속 단계로 분리한다.
- **총 예상 규모**: 중 (3~4주) — Phase 1 MVP 기준. 고급 분석 포함 시 대(1개월 이상).

## 5. 항목별 최종 실행 계획

### 0-1. 상위 목적 문장 + 장기 성장 정의

- **목표**: 돌로무드를 오래 쓰면 사용자의 삶에서 무엇이 달라지는지 한 문장으로 정의. 이후 모든 UX 카피·KPI·브랜드 톤의 상위 기준이 됨.
- **구체적 행동**:
  1. 기획자가 후보 문장 3개 + 장기 성장 서사(6개월·1년 사용 시 기대 변화) 초안 작성 (1일)
  2. 경영진/브랜드 담당과 30분 리뷰 미팅 → 1개 확정 (1일)
  3. 확정 문장을 제품 위키에 등록, 이후 모든 기획서에 참조 링크 포함
- **필요 산출물**: 목적 문장 후보 문서, 장기 성장 서사 1페이지
- **성공 기준**: 경영진 서면 승인 완료, 팀 전원이 목적 문장을 동일하게 인용 가능
- **리스크**: 경영진 간 의견 불일치로 확정 지연 → 대응: 리뷰 미팅 전 발신자(피드백 작성자)에게 선호 방향 사전 확인
- **모호한 부분**: 발신자의 정확한 승인 권한. 기획자가 확인 필요.

### 0-2. 감정 데이터 구조 기술 검증

- **목표**: 현재 감정 기록 데이터 스키마로 "변화 요약"을 어느 수준까지 구현 가능한지 확인.
- **구체적 행동**:
  1. 개발팀이 현재 감정 기록 테이블/컬렉션 스키마 공유 (0.5일)
  2. 기획자와 함께 "최근 N회 기록 비교"로 만들 수 있는 변화 문장 목록 도출 (0.5일)
  3. 불가능한 항목(회복 속도, 패턴 고급 분석) 명시적 분리 → Phase 3로 이관
- **필요 산출물**: 기술 검증 결과 문서 (가능/불가능/조건부 항목 표)
- **성공 기준**: MVP에서 구현할 변화 요약 범위가 기획-개발 간 합의됨
- **리스크**: 현재 스키마가 감정 강도 수치를 저장하지 않을 경우 → 대응: 자기보고 감정 라벨(카테고리)만으로 "가장 자주 느낀 감정 변화" 수준 요약
- **모호한 부분**: 비교 기준 시점 로직 (매일 사용자 vs 3일/1주 간격 사용자). 개발팀과 협의 필요.

### 0-3. 프로토타입 검증 설계

- **목표**: Phase 2에서 실행할 사용자 테스트의 프로토콜을 사전 확정하여, Phase 1 빌드와 병행 준비 가능하게 함.
- **구체적 행동**:
  1. 검증 목표 정의: ① 변화 요약을 보고 자기 변화를 이해했는가, ② 콘텐츠 추천 이유를 납득했는가, ③ 빠른 기록 사용자가 방해받지 않았는가, ④ 인지·정서 변화 간이 측정
  2. 설문 도구 선정: 감정 자기효능감 5문항 (사전/사후) — 기존 검증 척도(DERS 단축형 등) 활용 검토
  3. 태스크 목록 작성: 6~8개 구체 시나리오
  4. 표본 계획: 5~8명 (기존 사용자 3~4명 + 신규 2~3명 + 휴면 복귀 1~2명)
- **필요 산출물**: 검증 프로토콜 문서, 설문지, 태스크 시트, 리크루팅 기준
- **성공 기준**: 프로토콜 문서가 리서치 담당자 리뷰를 통과, Phase 1 완료 즉시 테스트 착수 가능
- **리스크**: UX 리서치 인력 부재 → 대응: 기획자가 직접 진행 가능한 간이 프로토콜로 설계. 임상심리 자문은 설문 도구 선정 시 1회 자문으로 커버
- **모호한 부분**: 임상심리 자문 확보 경로. 외부 파트너 또는 사내 자문인 필요.

### 1-1. 변화 요약 카드 설계 + 구현 (MVP)

- **목표**: 재방문 사용자가 홈 화면 최상단에서 "내 감정이 어떻게 달라졌는지"를 한 문장으로 확인할 수 있게 함.
- **구체적 행동**:
  1. 기획자: 변화 요약 문장 템플릿 정의 (3일)
     - 긍정적 변화: "최근 기록에서 {{감정A}}가 줄고, {{감정B}}가 늘고 있어요"
     - 안정적 유지: "최근 감정 패턴이 안정적이에요"
     - 부정적 변화: "최근 {{감정A}}가 자주 나타나고 있어요. 괜찮아요, 알아차린 것 자체가 중요해요" (톤 주의)
     - 데이터 부족: "조금 더 기록하면 변화를 보여드릴게요" (기록 3회 미만)
  2. 디자이너: 홈 상단 카드 UI 시안 (2일) — 문장 + 미니 감정 트렌드(최근 5~7회) 시각화
  3. 개발자: 최근 N회 기록 비교 로직 + 변화 요약 API + 프론트엔드 (5일)
  4. QA: 다양한 기록 빈도 시나리오 테스트 (2일)
- **필요 산출물**: 변화 요약 문장 템플릿 시트, 카드 UI 디자인 시안, API 스펙
- **성공 기준**: ① 기록 3회 이상 재방문 사용자에게 변화 요약 카드 노출, ② 사용자 테스트에서 5명 중 4명 이상이 변화 메시지 의미를 올바르게 설명 가능
- **리스크**:
  - 부정적 변화 표시 시 사용자 불쾌감 → 대응: 카피 톤을 "알아차림" 프레이밍으로 통일, 사용자 테스트에서 톤 검증
  - 비교 기준 시점이 사용자마다 다름 → 대응: MVP는 "최근 N회" 고정 (N=5~7), 날짜 기반 비교는 Phase 3
- **모호한 부분**:
  - "명확히 보이도록"의 UI 형태 — 홈 상단 카드로 해석함. 풀스크린 인터스티셜은 기각 (빠른 기록 니즈 방해)
  - 부정적 변화도 보여줘야 하는지 — **보여주되 톤을 조절**하는 방향으로 해석함. 기획자 확인 필요.

### 1-2. 콜드스타트·빈 상태·악화 상태 UX

- **목표**: 데이터 부족·API 실패·감정 악화 등 예외 상황에서도 재방문 경험이 끊기지 않게 함.
- **구체적 행동**:
  1. 기획자: 상태별 화면 시나리오 정의 (1일)
     - 기록 0회: 첫 기록 유도 메시지
     - 기록 1~2회: "한 번 더 기록하면 변화를 보여드릴게요" + 지난 기록 리마인드
     - 7일 이상 휴면 복귀: "다시 돌아왔군요" + 마지막 기록 요약
     - API 실패/오프라인: 마지막 캐시된 요약 표시 또는 기본 메시지
     - 하루 다회 기록: 가장 최근 기록 기준 요약
  2. 디자이너: 각 상태별 UI 변형 시안 (2일)
  3. 개발자: 상태 분기 로직 구현 (1-1과 병행)
- **필요 산출물**: 상태별 시나리오 문서, UI 변형 시안
- **성공 기준**: 모든 엣지 케이스에서 빈 화면·에러 화면 없이 의미 있는 메시지 표시
- **리스크**: 상태 조합이 많아 QA 부담 증가 → 대응: 핵심 3가지(데이터 부족, 악화, 오프라인)만 MVP, 나머지는 기본 fallback
- **모호한 부분**: 감정 악화 시 전문 도움 연결 필요 여부 → 법무/임상 자문 확인 대상

### 1-3. 콘텐츠 노출 순서 재배치 + skip 정책

- **목표**: 재방문 화면에서 "변화 확인 → 콘텐츠"의 순서를 확립하되, 빠른 기록 사용자 경험을 보호.
- **구체적 행동**:
  1. 기획자: 홈 IA(정보 구조) 재설계 (2일)
     - 상단: 변화 요약 카드
     - 중단: 맥락 기반 콘텐츠 추천 (추천 이유 태그 포함)
     - 하단/FAB: 빠른 기록 버튼 (항상 접근 가능)
     - deep link 유입 시: 목적 화면 직행 후 홈 복귀 시 변화 요약 노출
  2. 개발자: 홈 화면 레이아웃 변경 + deep link 분기 (3일)
- **필요 산출물**: 홈 IA 와이어프레임, deep link 시나리오 문서
- **성공 기준**: ① 변화 요약이 콘텐츠보다 시각적으로 상위 배치, ② 빠른 기록 버튼 접근에 추가 탭 불필요, ③ 재배치 후 세션당 기록 횟수 감소 없음
- **리스크**: 변화 요약이 강제 게이트처럼 느껴짐 → 대응: 스크롤 자연 노출, 강제 인터스티셜 아님. skip 별도 조작 불필요
- **모호한 부분**: "반드시" 순서인데 스킵 가능하게 한 해석이 원문 의도와 부합하는지 → 자연 노출 + 상단 배치로 "보게 되는" 구조가 원문의 실질 의도라고 해석함. 기획자가 발신자에게 확인 권장.

### 1-4. 콘텐츠 추천 이유 문구 시스템

- **목표**: 모든 추천 콘텐츠에 "왜 이걸 보여주는지" 한 줄 맥락이 붙어, 콘텐츠가 감정 관리 도구로 인식되게 함.
- **구체적 행동**:
  1. 기획자: 콘텐츠 taxonomy 정의 — 감정 카테고리 5~8개 × 콘텐츠 유형 매핑표 (2일)
  2. 기획자: 추천 이유 템플릿 문구 작성 (1일)
     - 감정 상태 연결형: "{{감정}} 상태일 때 도움이 되는 활동이에요"
     - 기록 맥락 연결형: "방금 기록한 내용과 관련된 활동이에요"
     - 일반 맥락형(fallback): "기록 후 마음을 정리하기 좋은 활동이에요"
  3. 기획자 + 콘텐츠 운영: 기존 콘텐츠 태깅 (2일)
  4. 개발자: 콘텐츠 카드에 추천 이유 필드 추가 + 매칭 로직 (2일)
- **필요 산출물**: 콘텐츠 taxonomy 표, 템플릿 문구 시트, 태깅 가이드
- **성공 기준**: ① 전체 콘텐츠의 80% 이상에 추천 이유 문구 표시, ② 사용자 테스트에서 "왜 이 콘텐츠가 나왔는지 알겠다"라고 답하는 비율 > 70%
- **리스크**:
  - 오추천 시 신뢰 훼손 → 대응: MVP에서는 안전한 일반형 문구 위주, 구체적 감정 매칭은 정확도 검증 후 확대
  - 태깅되지 않은 콘텐츠 → 대응: fallback 문구 사용, 미태깅 콘텐츠 우선순위 낮춤
- **모호한 부분**: 오추천 문의/불만 대응 프로세스. CS팀과 사전 공유 필요.

### 1-5. 돌탑/친밀도 → 능력 축적 기본 메시지

- **목표**: 돌탑 성장·친밀도 상승이 단순 시각 보상이 아니라 "사용자의 성장"을 의미함을 전달.
- **구체적 행동**:
  1. 기획자: 기록 횟수 기반 프록시 메시지 초안 작성 (1일)
     - 예: "N번의 기록을 통해 당신은 자신의 감정을 더 자주 들여다보게 되었어요"
     - 예: "감정을 기록하는 습관이 자리 잡고 있어요"
     - 지양: "당신이 감정 조절을 더 잘하게 되었습니다" (치료적 효능 주장 리스크)
  2. 법무 검토: 능력 축적 표현이 의료적/치료적 효과 주장으로 읽히지 않는지 확인 (1일)
  3. 디자이너: 돌탑 성장 화면에 메시지 배치 시안 (1일)
  4. 개발자: 돌탑 성장 이벤트 시 메시지 표시 로직 (1일)
- **필요 산출물**: 메시지 카피 시트 (법무 검토 완료 버전), UI 시안
- **성공 기준**: ① 돌탑 성장 시 능력 축적 메시지 1종 이상 표시, ② 사용자 테스트에서 메시지를 "설교적"으로 느낀 응답 0건, ③ 법무 검토 통과
- **리스크**: Z세대 톤과 불일치 → 대응: 사용자 테스트에서 카피 톤 A/B 검증 (부드러운 톤 vs 위트 있는 톤)
- **모호한 부분**: 4가지 능력(알아차림 속도, 강도 안정화, 회복력, 패턴 인식) 중 어떤 것을 프록시로 선택할지 → 0-1의 목적 문장 확정 후 기준 설정. 기획자 판단 사항.

### 1-6. 감정돌 확장 설계 철학 문서

- **목표**: 감정돌이 장기적으로 '감정 상태 직관 인터페이스'로 확장될 수 있다는 설계 방향을 명문화.
- **구체적 행동**:
  1. 기획자: 1페이지 설계 철학 문서 작성 (0.5일)
     - 현재: 감정을 가볍게 만드는 캐릭터
     - 미래: 감정 상태를 직관적으로 보여주는 인터페이스 (VAD, 워치 연계 가능성)
     - 원칙: 캐릭터의 정서적 유대를 유지하면서 데이터 레이어를 점진적으로 추가
  2. 디자이너·개발 리드에게 공유 및 코멘트 수렴 (0.5일)
- **필요 산출물**: 감정돌 확장 설계 철학 1페이지 문서
- **성공 기준**: 기획·디자인·개발 리드 3자 리뷰 완료, 코멘트 반영
- **리스크**: 낮음. 구현이 아닌 방향 정렬이므로.
- **모호한 부분**: VAD/워치 연계의 구체적 타임라인 — 이 문서에서는 다루지 않음(Phase 3 이후).

### 1-7. 민감정보·표현 리스크 검토

- **목표**: 감정 데이터 처리와 능력 축적 메시지가 법적·윤리적 기준을 충족하는지 사전 확인.
- **구체적 행동**:
  1. 기획자: 검토 요청서 작성 — 능력 축적 메시지 후보, 감정 데이터 활용 범위, 워치 데이터 수집 가능성 (0.5일)
  2. 법무/개인정보 담당: 검토 및 가이드라인 회신 (2~3일)
- **필요 산출물**: 표현 가이드라인 (가능/불가능 표현 목록), 감정 데이터 처리 기준 확인서
- **성공 기준**: 법무 검토 완료, 능력 축적 메시지 표현 범위 확정
- **리스크**: 법무 검토 지연 → 대응: 1-5 메시지를 가장 안전한 수준(기록 횟수 기반)으로 한정하여 병행 진행
- **모호한 부분**: 미성년 사용자 존재 시 추가 동의 필요 여부

### 2-1~2-3. Phase 2 검증 (통합)

- **목표**: MVP가 원문의 핵심 요구("변화 인식 중심 재방문 경험")를 실제로 달성하는지 증명.
- **구체적 행동**:
  1. 사용자 테스트 (리서치 + 기획, 3~4일)
     - 대상: 5~8명 (기존 3~4 / 신규 2~3 / 휴면 복귀 1~2)
     - 태스크 목록:
       - T1: 앱을 열고 가장 먼저 눈에 띄는 것을 말해주세요
       - T2: 변화 요약 카드를 보고 무슨 의미인지 설명해주세요
       - T3: 추천된 콘텐츠가 왜 나왔는지 추측해주세요
       - T4: 감정 기록을 빠르게 하고 싶을 때 어떻게 하시겠어요?
       - T5: 돌탑이 자랐을 때 나오는 메시지를 읽고 느낌을 말해주세요
       - T6: (사후) 이 앱을 다시 열고 싶은 이유가 있다면?
  2. 인지·정서 간이 설문 (리서치, 2일)
     - 사전: "감정을 마주하는 것이 부담스럽다" 등 5문항 (7점 척도)
     - 사후 (1주 사용 후): 동일 5문항 재측정 + 자유 응답
     - 정성 인터뷰: 3~5명 30분 심층 — "이 앱을 쓰면서 달라진 점이 있나요?"
  3. 정량 지표 수집 (데이터, 1주 모니터링)
     - A군(변화 요약 있음) vs B군(기존 홈): 7일 리텐션, 세션 시간, 기록 빈도
     - 변화 요약 카드 체류 시간 (자동 노출 vs 의도적 조회 구분)
     - 콘텐츠 추천 이유 노출 후 전환율 vs 미노출
- **필요 산출물**: 사용자 테스트 결과 보고서, 설문 분석 결과, A/B 데이터 리포트
- **성공 기준**:
  - 사용자 5명 중 4명 이상이 변화 요약 의미를 올바르게 설명
  - "다시 열고 싶은 이유"에서 '변화 확인/궁금함' 관련 답변 > 50%
  - A군 7일 리텐션이 B군 대비 하락하지 않음 (최소 동등)
  - 사전/사후 설문에서 감정 부담 점수 감소 경향 관찰 (통계적 유의성은 표본 한계로 기대하지 않음)
- **리스크**: 표본이 작아 정량 결론 도출 어려움 → 대응: 정성 인사이트를 우선하되, 정량은 방향성 확인용으로만 활용
- **모호한 부분**: A/B 테스트 인프라 존재 여부. 없을 경우 코호트 비교로 대체.

| 행동 지표 | 콘텐츠 전환율 (추천 이유 노출 시) | 미측정 | 미노출 대비 +15% | A/B 비교 |
| 행동 지표 | 세션당 기록 횟수 | 미측정 (베이스라인) | 기존 대비 감소 없음 | 앱 분석 도구 |
| 인지 지표 | 변화 메시지 이해도 | 없음 | 사용자 테스트 5명 중 4명 정확 설명 | 태스크 기반 사용자 테스트 |
| 인지 지표 | "다시 올 이유" 중 변화 확인 비율 | 없음 | 응답의 50% 이상 | 사후 인터뷰 키워드 분석 |
| 정서 지표 | 감정 부담 점수 | 사전 측정 | 사후 감소 경향 | 5문항 7점 척도 설문 (사전/사후) |
| 정서 지표 | 능력 축적 메시지 수용도 | 없음 | "설교적" 응답 0건 | 사용자 테스트 정성 피드백 |

## 7. 검증 계획

### 프로토타입 검증
- **대상**: Figma 프로토타입 또는 개발 빌드 (Phase 1 산출물)
- **방법**: 태스크 기반 사용성 테스트 (5~8명, 개별 30~40분)
- **핵심 검증 질문**:
  1. 사용자가 변화 요약을 자기 상태 변화로 이해하는가?
  2. 콘텐츠 추천 이유가 납득되는가?
  3. 빠른 기록 흐름이 방해받지 않는가?
  4. 능력 축적 메시지가 자연스러운가?

### 사용자 테스트 시나리오 (구체 태스크)
1. 앱 홈 진입 → 첫 화면에서 가장 눈에 띄는 정보 설명
2. 변화 요약 카드 → 내용 해석 및 자기 감정과 연결 여부
3. 콘텐츠 추천 영역 → 왜 이 콘텐츠가 나왔는지 추측
4. 빠른 감정 기록 → 기록까지의 탭 수와 소요 시간
5. 돌탑 성장 메시지 → 메시지 의미 해석과 감정적 반응
6. (사후) 이 앱을 다시 열고 싶은 이유가 있다면?

### A/B 테스트 설계 (Phase 2 정량)
- **실험군**: 변화 요약 카드 + 추천 이유 문구 + 능력 축적 메시지 적용
- **대조군**: 기존 홈 화면
- **주요 지표**: 7일 리텐션, 세션 시간, 기록 빈도, 콘텐츠 전환율
- **기간**: 최소 7일 (표본 허용 시 14일)
- **주의**: A/B 인프라 미확인. 불가 시 시간 기반 코호트 비교로 대체.

### 정량 + 정성 측정 항목
| 측정 유형 | 항목 | 도구 | 시점 |
|----------|------|------|------|
| 정량-행동 | 재방문율, 세션 시간, 기록 빈도 | Firebase/Amplitude | Phase 2 전 기간 |
| 정량-인지 | 감정 부담 5문항 사전/사후 | 설문 (앱 내 또는 외부) | Phase 2 시작/종료 |
| 정성 | 변화 메시지 이해도, 톤 수용도 | 사용자 테스트 + 인터뷰 | Phase 2 중 |

## 9. 기획자가 첫 3일 안에 할 것

### 가장 먼저 확정해야 할 결정 (3개)
1. **상위 목적 문장 후보 3개 작성 + 발신자에게 선호 방향 사전 확인** — 이것이 모든 UX 카피, 능력 축적 메시지, 검증 기준의 상위 원칙이 됨
2. **개발팀과 감정 데이터 스키마 확인** — 현재 저장되는 필드(감정 라벨, 강도 수치, 타임스탬프 등)를 파악해야 변화 요약 MVP 범위가 나옴
3. **"반드시 순서"의 강제 수준을 발신자에게 확인** — 자연 노출(스크롤 상단 배치)로 충분한지, 인터스티셜 수준의 강제가 필요한지

### 가장 먼저 만들어야 할 산출물
- **변화 요약 문장 템플릿 초안** (긍정/안정/부정/데이터 부족 4가지 케이스) — 이것이 있어야 디자이너가 카드 시안을, 개발자가 API 스펙을 시작할 수 있음

### 가장 먼저 확인해야 할 가정
- **"사용자는 자기 감정의 변화를 확인하러 앱을 다시 연다"** — 이것이 전체 재설계의 핵심 가정. 현재 직접 검증 데이터가 없으므로, Phase 2 사용자 테스트에서 반드시 검증해야 함. 만약 이 가정이 틀리면 전체 방향 재검토 필요.

### 반드시 싱크해야 할 이해관계자
1. **발신자(피드백 작성자)** — 상위 목적 문장 선호 방향, 순서 강제 수준, 부정적 변화 표시 정책 확인
2. **개발 리드** — 감정 데이터 스키마, 변화 요약 MVP 기술 가능성, A/B 인프라
3. **법무/개인정보 담당** — 능력 축적 메시지 표현 범위, 감정 데이터 활용 기준, 미성년자 이슈
