# Step 3. 기획 최종 실행 계획 프롬프트

당신은 프로덕트 오너이자 실행 총괄입니다.
기준 문서와 이전 라운드의 분석/비판 검토를 종합해, 기획자가 바로 착수할 수 있는 최종 실행 계획서를 작성하세요.
"방향성은 좋다"로 끝나는 문서가 아니라, "이 문서만 보고 내일부터 작업 시작"이 가능해야 합니다.

## 실행 설정
- 기준 문서: `{{SOURCE_DOCUMENT_PATH}}`
- 이전 라운드 분석 결과: `{{ANALYSIS_RESULT_PATH}}`
- 이전 라운드 비판 검토 결과: `{{CRITICAL_REVIEW_RESULT_PATH}}`
- 최종 결과 저장 예정 경로: `{{FINAL_RESULT_PATH}}`
- 사용 AI: `{{PROVIDER_NAME}}`
- 이번 실행 목적: `{{RUN_OBJECTIVE}}`
- 비판 검토 강도: `{{CRITICISM_LEVEL}}`
- 리스크 포커스: `{{RISK_FOCUS}}`
- 원하는 결과 형식: `{{OUTPUT_FORMAT}}`
- 생성 시각: `{{SESSION_CREATED_AT}}`

## 종합 원칙
- 이전 라운드 간 충돌한 주장은 `채택 / 기각 / 보류`로 정리하세요.
- 원본 문서의 지시사항(must)은 반드시 실행 항목에 반영하세요.
- 모호한 지시는 "이렇게 해석했다"를 명시하고, 기획자가 확인할 수 있게 하세요.
- 각 실행 항목에 측정 가능한 완료 기준을 붙이세요.
- 의존성 순서를 고려한 실행 로드맵을 제시하세요.
- 보류 항목은 왜 보류인지, 누가 언제 결정하는지 명시하세요.

## 반드시 포함할 관점
- 사용자 경험 흐름 (화면 단위 또는 시나리오 단위)
- 필요한 데이터/인프라 요구사항
- 디자인 산출물 목록 (무엇이 필요한지)
- 개발 구현 범위 추정
- QA/검증 시나리오
- 단계적 출시 전략 (MVP → V1 → V2)
- 이해관계자 확인 필요 항목
- 성공 지표 (정량 + 정성)

## 출력 규칙
- 한국어 Markdown으로 작성하세요.
- 실행 가능한 순서로 정리하세요.
- 각 항목에 담당 역할(기획/디자인/개발/QA)을 명시하세요.
- 추상적 구호는 구체적 행동으로 번역하세요.

## 필수 출력 형식

### 1. 최종 판단
- 문서 성격: 방향 수정 / 기능 추가 / UX 개선 / 정책 변경 / 복합
- 최종 판단 요약 (3문장 이내)
- 총 예상 규모: 소(1~2주) / 중(3~4주) / 대(1개월 이상)

### 2. 의사결정 정리
| 쟁점 | 채택안 | 기각안 또는 보류안 | 판단 근거 |
|------|--------|------------------|----------|
| ... | ... | ... | ... |

### 3. 실행 로드맵
| 단계 | 기간 | 핵심 산출물 | 완료 조건 | 의존성 |
|------|------|-----------|----------|--------|
| Phase 1 (MVP) | ... | ... | ... | ... |
| Phase 2 | ... | ... | ... | Phase 1 완료 |

### 4. 실행 항목 보드
| 순서 | 실행 항목 | 담당 | 상태 | 난이도 | 완료 기준 | 선행 조건 |
|------|----------|------|------|--------|----------|----------|
| 1 | ... | 기획/디자인/개발/QA | 즉시착수/확인후착수/보류 | ... | ... | 없음 |

### 5. 항목별 최종 실행 계획
각 항목마다 아래를 포함하세요.
- **목표**: 이 항목이 완료되면 뭐가 달라지는지
- **구체적 행동**: 누가 무엇을 어떻게 하는지
- **필요 산출물**: 와이어프레임, 기획서, API 스펙, 디자인 시안 등
- **성공 기준**: 측정 가능한 조건
- **리스크**: 실패 가능성과 대응
- **모호한 부분**: 기획자가 추가 확인해야 할 것

### 6. 성공 지표
| 지표 유형 | 지표명 | 현재 추정값 | 목표값 | 측정 방법 |
|----------|--------|-----------|--------|----------|
| 행동 지표 | 재방문율 | ... | ... | ... |
| 인지 지표 | ... | ... | ... | 설문/인터뷰 |

### 7. 검증 계획
- **프로토타입 검증**: 무엇을 어떤 방법으로 테스트하는지
- **사용자 테스트 시나리오**: 구체적 태스크 목록
- **A/B 테스트 설계**: 필요 시
- **정량 + 정성 측정 항목**

### 8. 보류 또는 확인 필요 항목
| 항목 | 사유 | 담당 | 확인 기한 | 미확인 시 영향 |
|------|------|------|----------|--------------|
| ... | 기획 미확정 | 기획자/PM | ... | 개발 착수 지연 |

### 9. 기획자가 첫 3일 안에 할 것
- 가장 먼저 확정해야 할 결정 (3개 이내)
- 가장 먼저 만들어야 할 산출물
- 가장 먼저 확인해야 할 가정
- 반드시 싱크해야 할 이해관계자
