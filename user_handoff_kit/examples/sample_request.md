# 예시 입력 문서

## 작업 제목
- 프로필 편집 화면에서 닉네임 저장 후 성공 토스트가 두 번 노출되는 문제 수정

## 작업 유형
- 버그 수정

## 작업 배경
- 사용자 피드백에서 저장 성공 후 동일한 토스트가 연속 두 번 보인다는 제보가 들어왔습니다.
- 운영팀은 API 중복 호출인지, 화면 상태 중복 처리인지 아직 판단하지 못했습니다.

## 목표
- 성공 토스트는 한 번만 노출되어야 합니다.
- 저장 성공 이후 화면 상태와 뒤로가기 흐름은 기존과 동일해야 합니다.

## 사용자 시나리오
1. 사용자가 프로필 편집 화면에 진입합니다.
2. 닉네임을 수정하고 저장 버튼을 누릅니다.
3. 저장 성공 후 성공 토스트가 한 번만 노출됩니다.

## 상세 요구사항
- 실패 토스트 정책은 유지합니다.
- 저장 중 로딩 UI가 있다면 기존 동작을 깨지 않아야 합니다.
- 마이페이지와 회원가입 프로필 단계가 공통 컴포넌트를 공유하는지도 확인해야 합니다.

## 예상 영향 범위
- 프로필 편집 화면
- 공통 토스트 호출부
- 상태 관리
- API 응답 처리

## 완료 조건
- 성공 토스트가 한 번만 노출됩니다.
- 동일 정책을 쓰는 다른 화면에서 회귀가 없어야 합니다.

## 확인 필요
- 회원가입 프로필 단계도 동일 컴포넌트를 사용하는지
- 중복 토스트가 화면단 문제인지 상태단 문제인지
