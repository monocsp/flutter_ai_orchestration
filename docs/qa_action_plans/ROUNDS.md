# Codex ↔ Claude 협업 3라운드 정리

## 라운드 1: 초기 분류/원인 가설 정리
- Codex 입력
  - QA 체크리스트와 Jira 이슈를 기반으로 실패 항목 중심으로 분류
  - 코드베이스에서 이슈별 후보 파일 탐색
- Claude 산출(기획 관점)
  - 이슈별 우선순위: `QA-117 > TIET-68 > TIET-69 > QA-116 > RTXG-2 > XAMO-286(모니터링)`
  - 사용자 체감 영향 기준으로 BGM/회원가입 검증/닉네임 정책을 상위 배치

## 라운드 2: 구현 단위/검증 기준 구체화
- Codex 입력
  - 이슈별 관련 함수/화면 경로를 구체화
  - 공통 컴포넌트 영향 범위(`fm_snackbar`, `UserInteraction`) 도출
- Claude 산출(리뷰 관점)
  - 공통 로직 변경 시 회귀 위험 지점 명시
  - 각 이슈 완료 기준을 `재현 시나리오 + 차단 조건 + 문구 일치`로 정의

## 라운드 3: 실행 계획 확정
- Codex 입력
  - 이슈별 액션 플랜 md 초안 통합
  - 단계별 적용 순서와 의존성(공통 로직 우선)을 제시
- Claude 산출(최종 합의)
  - 실행 순서 확정
    1. `QA-117` (BGM 정책/라우트 동기화)
    2. `TIET-68` (회원가입 실시간 검증/성공 메시지)
    3. `TIET-69` (닉네임 정책 공통화)
    4. `QA-116` (복귀 간헐 케이스 안정화)
    5. `RTXG-2` (적용 범위 누락 보강)
    6. `XAMO-286` (회귀 모니터링)
  - 문서 산출물 기준으로 개발 착수 가능 상태 확인

## 최종 산출물
- 이슈별 계획서 6개
- 협업 라운드 문서 1개
- 공통 인덱스 문서 1개
