# QA 이슈 액션 플랜 패키지

## 목적
QA 체크리스트 결과와 Jira 코멘트를 기준으로, 이슈별 수정 계획과 코드 연관 파일을 문서화합니다.

## 문서 목록
- `TIET-68.md`: 회원가입 이메일/비밀번호 검증 및 성공 메시지 노출
- `TIET-69.md`: 닉네임 검증 토스트 정책 일관성(회원가입 vs 마이페이지)
- `XAMO-286.md`: 감정 피드백 표정 크기/비율 일관성
- `QA-116.md`: 백그라운드 복귀 시 로딩/스낵바/진행률 복원
- `QA-117.md`: 백그라운드 복귀 및 화면 전환 시 BGM 정확성
- `RTXG-2.md`: 스낵바 수직 스와이프 dismiss 적용 범위/재노출
- `ROUNDS.md`: Codex ↔ Claude 협업 3라운드 정리

## 공통 작업 원칙
- 변경 범위는 최소화하고, 이슈별 핵심 경로만 수정합니다.
- `lib/core` 공통 컴포넌트 변경 시 영향 범위를 반드시 재검증합니다.
- 재현이 간헐적인 이슈는 로깅 포인트를 먼저 추가하고 정책 정의를 확정합니다.
- 모든 이슈는 `flutter analyze` 및 대상 플로우 수동 QA를 통과해야 완료 처리합니다.
