# QA-117 액션 플랜

- Jira: https://dolomood.atlassian.net/browse/QA-117
- 주제: 백그라운드 복귀 시 BGM 유지 정책

## QA 결과 요약
- 실패(⚠️)
  - 타로 화면 복귀 시 홈 BGM/타로 BGM 정책 불일치
  - 홈↔타로 전환 후 복귀 시 현재 화면 기준 BGM 오재생
  - 복귀 직후 지연 시작(2~3초) 체감, 중첩/끊김 정책 불명확

## 원인 가설
- 앱 라이프사이클 resume 시 `UserInteraction().playBGM()`가 마지막 상태를 재생하지만, 라우트 기준 소스와 동기화가 늦거나 누락
- `changedBGM` 호출 지점(홈/타로 진입·이탈)과 resume 호출 타이밍 경합
- 지연 시작 허용 범위가 정책으로 정의되지 않아 QA 판단 기준이 흔들림

## 수정 계획
1. BGM 소스 결정 우선순위 명확화
- `현재 라우트 기반 목표 BGM`을 1순위, `currentPlayingBgm`은 2순위 fallback으로 정리합니다.

2. resume 시 라우트 동기화 보강
- 앱 복귀 시점에 현재 화면 라우트를 재평가해 `changedBGM`을 재호출하거나 동기화 함수를 둡니다.

3. 중복 재생/지연 시작 제어
- 동일 BGM 재생 요청은 early return으로 차단하고, 전환 시 stop→play 순서 보장으로 중첩을 방지합니다.
- `복귀 후 시작 지연 허용 기준(ms)`을 정책으로 정의하고 QA 기준을 문서화합니다.

4. 홈/타로 왕복 회귀 케이스 추가
- 홈→타로→홈→백그라운드→복귀 순서를 자동/수동 시나리오로 고정합니다.

## 검증 시나리오
- 홈 화면에서 백그라운드→복귀: 홈 BGM 유지
- 타로 화면에서 백그라운드→복귀: 타로 BGM 유지
- 타로→홈 전환 후 복귀: 홈 BGM 재생
- 복귀 직후 5회 반복에서 중첩/끊김/과도 지연 미발생

## 완료 기준
- QA-117 체크리스트 3개 항목 모두 ✅
- BGM 지연 허용 정책이 문서화되어 QA/개발 판단 기준 일치
