# TIET-68 액션 플랜

- Jira: https://dolomood.atlassian.net/browse/TIET-68
- 주제: 회원가입 이메일/비밀번호 검증 및 다음 버튼 동작

## QA 결과 요약
- 성공(✅)
  - 잘못된 이메일 형식 에러(빨강) 노출
  - 이메일 중복 체크 실패 시 다음 버튼 비활성
  - 비밀번호 정책 미충족 시 버튼 차단 동작
  - 비밀번호 확인 필드 포커스 시 조건부 버튼 활성
  - 유효하지 않은 상태에서 버튼 활성처럼 보이는 문제 없음
  - 재포커싱 시 성공/실패 컬러 역전 없음
- 실패(⚠️)
  - 이메일 유효 시 성공 메시지(`사용 가능한 이메일이에요`) 미노출
  - 비밀번호 정책 미충족 에러가 실시간 노출되지 않음
  - 비밀번호 확인 일치 시 성공 메시지(`비밀번호가 일치해요`) 미노출

## 원인 가설
- 이메일 성공 메시지 노출 타이밍이 포커스/중복체크 상태 갱신 순서에 의해 덮어씌워질 가능성
- 비밀번호 검증 메시지 갱신이 submit/포커스 이벤트 위주로만 실행되고 텍스트 변경 리스너 반영이 약함
- 확인 비밀번호 성공 메시지가 버튼 활성 조건과 분리되어 있어 특정 분기에서 비노출 가능성

## 관련 파일
- `lib/feature/signup/ui/widget/signup_email_password_widget.dart`
- `lib/feature/signup/ui/widget/signup/textfield/confirm_password_textfield_widget.dart`

## 수정 계획
1. 이메일 성공 메시지 상태 정규화
- 이메일 검증/중복체크 성공 시 단일 함수로 메시지/색상/notifier를 갱신합니다.
- 포커스 아웃/재포커스 시 메시지가 비정상 초기화되지 않도록 분기 우선순위를 정리합니다.

2. 비밀번호 실시간 검증 반영
- 비밀번호 입력 리스너에서 정책 검증 함수를 매 타이핑에 호출합니다.
- 정책 미충족 시 즉시 에러 문구 노출, 충족 시 에러 해제 처리합니다.

3. 확인 비밀번호 성공 메시지 노출 조건 통합
- `비밀번호 유효 + 확인값 일치`일 때 성공 메시지 노출을 단일 지점에서 보장합니다.
- 메시지 노출과 버튼 활성 조건을 동일 소스로 맞춥니다.

4. 회귀 방지
- 색상 역전/버튼 상태 불일치에 대한 수동 시나리오를 재실행합니다.

## 검증 시나리오
- 이메일 형식 유효 + 중복체크 성공 시 성공 메시지/초록 상태 확인
- 비밀번호 입력 중 정책 미충족 즉시 에러 노출 확인
- 확인 비밀번호 일치 즉시 성공 메시지 노출 확인
- 세 필드 포커스 이동 반복 시 메시지/버튼 상태 일관성 확인

## 완료 기준
- TIET-68의 ⚠️ 3개 항목이 모두 ✅로 전환
- 기존 ✅ 항목 회귀 없음
