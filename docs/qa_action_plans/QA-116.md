# QA-116 액션 플랜

- Jira: https://dolomood.atlassian.net/browse/QA-116
- 주제: 백그라운드 복귀 시 로딩/스낵바/진행률

## QA 결과 요약
- 성공(✅)
  - 진행률 % UI 유지
  - 오류 스낵바 노출 시간 정책 유지
  - 진행률/타이머 비정상 점프 없음
- 실패(⚠️)
  - 복귀 후 무한 로딩 고정 방지 및 재시도/홈 이동 플로우가 간헐적으로 불안정

## 원인 가설
- 복귀 이벤트 처리 시점에서 `retryCount`, 로딩 상태, 타이머 상태가 경합해 일부 분기가 누락될 가능성
- 복귀 직후 중복 트리거로 인해 스낵바와 진행 재개 로직이 엇갈리는 경우가 존재

## 관련 파일
- `lib/feature/tarot/ui/page/start/tarot_reading_loading_page.dart`

## 수정 계획
1. 복귀 처리 분기 순서 고정
- `_handleForegroundResume()`에서 상태 판별 순서를 명시적으로 정리합니다.
- `재시도 가능`, `재시도 한도 초과`, `정상 진행 재개` 분기를 상호배타적으로 보장합니다.

2. 타이머/진행률 재시작 idempotent 처리
- 이미 실행 중인 타이머가 있으면 중복 시작하지 않도록 가드합니다.

3. 오류 플로우 강제 보장
- 오류 상태에서 스낵바 액션(재시도/홈 이동)이 항상 표시되도록 조건을 재정렬합니다.

4. 간헐 재현용 로그 포인트 추가
- 복귀 시점 상태 스냅샷(로딩 상태, retryCount, 타이머 활성 여부)을 debug 로그로 남깁니다.

## 검증 시나리오
- 로딩 중 백그라운드↔복귀를 연속 10회 반복
- retryCount 0~3 구간별 복귀 동작 확인
- 복귀 직후 스낵바 액션 정상 동작(재시도/홈 이동) 확인

## 완료 기준
- 간헐 케이스 포함 복귀 테스트에서 무한 로딩 고정 재현 0건
